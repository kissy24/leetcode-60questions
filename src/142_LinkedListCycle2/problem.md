# 連結リストのサイクル検出 II

## 問題文

連結リストの先頭ノード `head` が与えられたとき、サイクルが始まるノードを返してください。サイクルが存在しない場合は `null` を返してください。

連結リストにサイクルが存在するとは、`next` ポインタを辿り続けることで再び到達できるノードが存在する場合を指します。内部的には、末尾ノードの `next` ポインタが接続されているノードのインデックス（0始まり）を `pos` で表します。サイクルが存在しない場合は `-1` となります。なお、`pos` はパラメータとして渡されません。

**連結リストを変更しないでください。**

---

## 例

**例 1:**

```
入力: head = [3,2,0,-4], pos = 1
出力: tail connects to node index 1
説明: 連結リストにサイクルが存在し、末尾は2番目のノード（インデックス1）に接続されています。
```

**例 2:**

```
入力: head = [1,2], pos = 0
出力: tail connects to node index 0
説明: 連結リストにサイクルが存在し、末尾は1番目のノード（インデックス0）に接続されています。
```

**例 3:**

```
入力: head = [1], pos = -1
出力: no cycle
説明: 連結リストにサイクルは存在しません。
```

---

## 制約

- リスト内のノード数は `[0, 10^4]` の範囲
- `-10^5 <= Node.val <= 10^5`
- `pos` は `-1` または連結リスト内の有効なインデックス
